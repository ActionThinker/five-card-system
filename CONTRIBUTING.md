# 贡献指南 · Contributing Guide

感谢你对五卡体系的关注！这是一套开源的结构化知识工具，我们欢迎各种形式的贡献。

---

## 贡献方式

### 1. 提交案例 · Submit a Case

五卡体系的真正价值来自真实世界的经验。如果你用五卡体系记录了你的经验，欢迎提交到社区。

**提交步骤：**

1. Fork 本仓库
2. 在 `examples/` 目录下创建一个新文件夹，以你的案例主题命名（如 `examples/客户续约谈判/`）
3. 在文件夹中放入你的案例卡、决策卡、方法论卡等（Markdown 格式）
4. 在 `examples/README.md` 的案例列表中加上你的案例链接
5. 提交 Pull Request

**案例提交要求：**
- 使用真实的经历（可以对敏感信息做脱敏处理，但请注明）
- 遵循手册中每张卡的字段格式
- 包含具体的数字和时间（不要只写模糊的描述）
- 标注是否已脱敏处理

**案例模板：** 参见 `examples/README.md` 中的模板。

---

### 2. 改进方法论 · Improve the Methodology

五卡体系本身也在不断进化。如果你对以下方面有建议：

- 某张卡片的字段设计
- 填写流程的优化
- 新的卡片类型的提议
- 画布布局的改进

请通过以下方式参与：

1. 在 GitHub Issues 中创建一个 `methodology` 标签的 Issue，描述你的建议
2. 说明建议背后的逻辑——最好能附带一个你遇到的真实场景，说明为什么现有设计不够用
3. 如果你有具体的修改方案，欢迎直接提 PR

**讨论原则：**
- 以真实案例为基础，而非纯理论推演
- 尊重体系的简洁性——五张卡是经过大量实践验证的最小完备集
- 说清楚"为什么需要改"比"怎么改"更重要

---

### 3. 翻译 · Translation

五卡体系目前以中文为主。如果你希望将其翻译成其他语言：

1. 在 Issues 中创建一个 `translation` 标签的 Issue，说明你想翻译的语言
2. 从 `handbook/` 中的章节开始翻译
3. 翻译时保留原文的术语一致性（如"决策卡"统一译为 "Decision Card"）
4. 保持 Markdown 格式不变

已计划的语言：English（进行中）、日本語（待开始）。

---

### 4. 报告问题 · Report Issues

如果你发现：
- 手册中的错误或不够清晰的地方
- 画布模板的问题
- 任何改进建议

请在 GitHub Issues 中提交，并：
- 使用清晰的标题
- 描述你遇到的具体问题
- 如果是内容问题，指出具体文件和段落
- 如果是建议，说明"为什么"

## 行为准则 · Code of Conduct

### 我们的承诺

为了营造一个开放和友好的环境，我们承诺：无论经验水平、性别、性别认同和表达、性取向、残疾、个人外貌、体型、种族、民族、年龄、宗教或国籍，所有参与五卡体系项目的人都应获得尊重和友好对待。

### 我们的标准

**鼓励的行为：**
- 使用友好和包容的语言
- 尊重不同的观点和经验
- 优雅地接受建设性批评
- 关注对社区最有利的事情
- 对其他社区成员表现出同理心

**不被容忍的行为：**
- 使用性化的语言或图像，以及不受欢迎的性关注或挑逗
- 挑衅、侮辱/贬损性评论，以及人身或政治攻击
- 公开或私下骚扰
- 未经明确许可发布他人的私人信息
- 在专业环境中被认为不适当的其他行为

### 我们的责任

项目维护者有责任澄清可接受行为的标准，并对任何不可接受的行为采取适当和公平的纠正措施。

## 许可

所有贡献将在 [CC BY-SA 4.0](./LICENSE) 许可下发布。提交贡献即表示你同意在此许可下发布你的内容。

> **一个问题**：五卡体系不是"我的方法论"，而是"我们共同的方法论基础设施"。每一次贡献——无论是一个案例、一个改进建议、还是一个翻译——都在让这座桥更宽一点：让人类经验到AI理解的桥。

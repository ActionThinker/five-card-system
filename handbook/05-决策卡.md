# 第五章：决策卡——把你的选择变成可复盘的判断

> **决策卡是什么？** 你在什么情况下做了什么选择——以及为什么。

五卡体系中，决策卡是"承上启下"的枢纽。案例告诉你发生了什么，方法告诉你别人可以怎么做——但只有决策卡告诉你**当时为什么那样选**。它是组织记忆中最难复制、也最有长期价值的部分。

---

## 一、为什么决策卡是"最难写也最重要"的卡？

答案很简单：**事后解释一个选择很容易，还原选择时的真实状态很难。**

你在2026年回头看2024年的一个决策，大脑会自动帮你"润色"——把当时的犹豫变成果断，把当时的运气变成远见，把当时忽视的风险变成"其实我早就想到了"。

决策卡的价值恰恰在于**对抗这种自动润色**。它强迫你回到选择发生的那一刻，还原你不知道结果时的真实思考。

> **决策卡的黄金法则**：写决策卡时，假装你不知道后来发生了什么。

## 二、决策卡的核心字段

### 2.1 基础信息

```markdown
## 基础信息
- **决策名称**: （一句话概括这个决策——"是否做X"或"在A和B之间选择"）
- **决策时间**: YYYY-MM-DD（越精确越好）
- **决策类型**: [战略选择 / 战术调整 / 资源分配 / 人事决定 / 产品方向 / 其他]
- **决策者**: 谁做的这个决定（可以是你自己，也可以是团队）
- **不可逆程度**: [可逆 / 部分可逆 / 不可逆]
```

### 2.2 当时的情境（最关键字段）

**这是决策卡的核心——还原你不知道结果时的状态。**

```markdown
## 当时的情境

**我（当时）知道什么：**
- 事实A
- 事实B
- （只写当时确实掌握的信息，不写后来才知道的）

**我（当时）不知道什么：**
- 不确定性X
- 不确定性Y
- （诚实列出当时没搞清的问题）

**时间压力：**
- 必须在什么时间前做决定？为什么？

**情绪状态（如果记得的话）：**
- 当时的主要情绪是？（焦虑/兴奋/犹豫/压力）
- 这个情绪有没有影响判断？
```

### 2.3 可选方案

列出你当时**认真考虑过**的所有选项——不只是你最终选的那个：

```markdown
## 可选方案

### 选项A: （名称）
- 预期收益
- 预期成本/风险
- 为什么认真考虑了这个选项

### 选项B: （名称）
- 预期收益
- 预期成本/风险
- 为什么认真考虑了这个选项
```

### 2.4 最终选择与拒绝理由

```markdown
## 最终选择

**我选择了：** 选项X

**我拒绝了：**
- 选项Y — 拒绝理由（不是"不好"，而是"在当时的约束下，不如X因为…"）
- 选项Z — 拒绝理由

**选择的核心逻辑（不超过3句话）：**
（如果你只能给未来的自己留3句话解释为什么这样选，你会写什么？）
```

### 2.5 实际结果

```markdown
## 实际结果

**短期结果（决策后3个月内）：**
- 发生了什么
- 和预期相比如何

**长期结果（决策后6个月以上）：**
- 发生了什么
- 有没有预期之外的影响

**结果评估：**
- [ ] 结果好 / [ ] 结果不好 / [ ] 结果好坏参半
- 即使结果好，回头看当时的决策逻辑对不对？（结果好 ≠ 决策对）
- 即使结果不好，回头看当时的决策逻辑对不对？（结果不好 ≠ 决策错）
```

### 2.6 复盘：学到了什么

```markdown
## 复盘

**决策逻辑的得失：**
- 当时的推理哪一步是对的？
- 当时的推理哪一步是错的？为什么错了？

**如果重来一次（带着当时的信息，不带事后结果）：**
- 还会做同样的选择吗？
- 如果不会，会在哪个环节做出不同的判断？

**可迁移的判断原则：**
- 从这个决策中可以提取出什么通用原则？
```

## 三、填写指南：还原你的决策时刻

### Step 1：选一个决策

不是所有"选择"都值得写决策卡。判断标准：

- ✅ 这个选择**有真实的不确定性**——你不知道结果会怎样
- ✅ 有**两个以上认真考虑过的选项**（只有一个选项不叫决策）
- ✅ 选择有**实质性后果**——选A和选B的区别足以影响项目的走向
- ❌ 选择是"显然的"——任何人在那个情境下都会这样选
- ❌ 选择太小——"今天午饭吃什么"不需要决策卡

### Step 2：回到"当时"

这是最关键、也最反直觉的一步。你需要**有意识地屏蔽事后信息**。

实用技巧：找一个你写决策卡时**还没有出现的结果**，用它来检验你有没有在"作弊"——如果你写"我选择A是因为后来证明A是对的"，那你就写错了。你应该写的是"我选择A是因为在当时的信息下，A的预期收益高于B，且A的可逆性更好"。

### Step 3：诚实列出"不知道的事"

这是决策卡最有价值的部分。列出你当时**确实不知道但很想知道**的信息。当你未来回头看这张卡时，这些"不知道"会让你重新审视：我当时是在多大程度的信息缺失下做的决定？

### Step 4：区分"结果好坏"和"决策对错"

这是决策卡最重要的一课：

| | 结果好 | 结果不好 |
|:---|:---|:---|
| **决策对** | 🏆 技能+运气 | 💔 正确的过程，糟糕的运气 |
| **决策错** | 🍀 运气好，救了一个坏决策 | 💀 最差的情况 |

> **不要因为结果好就认为决策正确。也不要因为结果不好就否定决策过程。** 区分"过程质量"和"结果质量"是做决策卡的核心能力。

### Step 5：提取可迁移的判断原则

最后问自己：从这个决策中，有没有一条可以带走的通用原则？不是"下次遇到X就选Y"（太具体），而是"当遇到某类情况时，用某种方式思考"。

## 四、完整示例：创始人决定是否转型

> **脱敏说明**：以下案例基于真实决策，行业、产品名称和金额已脱敏处理。

### 基础信息

| 维度 | 内容 |
|:---|:---|
| **决策名称** | 是否将产品从"通用项目管理工具"转型为"建筑行业专属方案" |
| **决策时间** | 2024年9月 |
| **决策类型** | 战略选择 / 产品方向 |
| **决策者** | 创始人（咨询了2位核心团队成员） |
| **不可逆程度** | 部分可逆——转型后可以再转回来，但会损失时间和市场窗口 |

### 当时的情境

**我（当时）知道什么：**
- 产品上线18个月，累计付费客户2,300家，但留存率只有34%（12个月后）
- 做了一个客户分层发现：建筑业客户的12个月留存率是67%，其他行业平均22%
- 建筑行业客户占付费客户的19%，但贡献了41%的收入
- 团队里有一个联合创始人有10年建筑行业背景
- 账上还有约600万，按当前烧钱速度能撑14个月

**我（当时）不知道什么：**
- 建筑行业的市场天花板有多大（做了初步测算但信心度只有60%）
- 聚焦一个行业后，其他行业的客户会流失多少（估计会流失但不知道程度）
- 竞争对手会不会也盯上建筑行业（没有任何情报）
- 转型需要多长时间才能在新定位下获得第一个客户

**时间压力：**
- 按当前的留存率，不改变的话可能在10-12个月后面临续约收入断崖
- 但也不是明天就必须做决定——有大约4-6周的窗口期做调研

**情绪状态：**
- 主要是焦虑。看着整体留存数据往下掉的感觉很难受。但同时建筑行业的数据又让人看到一线希望。担心"转型是逃避"——是不是因为不想面对产品本身的问题才想到转型？

### 可选方案

**选项A: 全面转型——砍掉非建筑行业功能，重新定位为建筑行业专属**

| 维度 | 评估 |
|:---|:---|
| 预期收益 | 建筑行业客户留存率可能进一步提升，获客更精准 |
| 预期成本 | 短期内流失约80%的非建筑行业客户，收入可能先降后升 |
| 风险 | 如果建筑业市场规模不够大，公司就没有退路了 |

**选项B: 双轨制——保持通用产品，但为建筑行业做深度定制**

| 维度 | 评估 |
|:---|:---|
| 预期收益 | 两边都不放弃，建筑行业获得更好的服务 |
| 预期成本 | 团队精力分散，14人团队做两条线可能是"两边都做不好" |
| 风险 | 资源分散导致两条线都做不出差异化 |

**选项C: 不做方向性调整——继续打磨通用产品，优化留存**

| 维度 | 评估 |
|:---|:---|
| 预期收益 | 投入在改善核心产品体验上，可能提升整体留存 |
| 预期成本 | 确定性最高，不需要承担转型风险 |
| 风险 | 如果留存率低不是产品体验问题而是"通用工具在垂直行业缺乏壁垒"，那投入就白费了 |

### 最终选择

**我选择了：** 选项A — 全面转型建筑行业。

**我拒绝了：**
- 选项B（双轨制）— 团队太小，14个人做双线就是两边都不够深。如果公司有40人，可能会选B。
- 选项C（不调整）— 很诱人，因为最安全。但直觉告诉我：如果问题本质是"通用工具对大多数行业没有足够价值"，再优化体验也不会改变留存率。我愿意用"可能犯错"换"可能做对"。

**选择的核心逻辑：**
1. 数据上：建筑行业的数据和其他行业的数据差距太大——这不是噪音，是信号。
2. 人上：团队有一个懂建筑行业的人——这是很多竞争对手没有的。
3. 时机上：现在还有14个月跑道，转型失败还有机会回头。再等半年，跑道缩短到8个月，就没有转型的勇气了。

### 实际结果

**短期结果（3个月内）：**
- 非建筑行业客户流失了约70%（比预期的80%好一点）
- 收入先跌了约25%，但第3个月开始回升
- 建筑行业新客户的获取周期缩短了40%（因为定位清晰了）

**长期结果（18个月后）：**
- 建筑行业客户从430家增长到1,200家
- 12个月留存率从67%提升到78%
- 收入恢复到转型前水平用了约8个月，之后超过了转型前最高点的1.8倍
- 意外收获：建筑行业内开始有口碑传播

**结果评估：**
- ✅ 结果好
- 但回头看：结果好有一定运气成分——如果建筑行业的数字化转型在那18个月里没有加速（恰好赶上了政策推动），转型的效果可能没有这么好。当时的决策逻辑中**低估了"行业趋势"这个外部变量**。

### 复盘

**决策逻辑的得失：**
- ✅ 对的：看到了数据中的信号（行业留存差异），没有把它当成噪声。
- ✅ 对的：考虑了团队能力匹配（有建筑行业背景的人），而不是只看市场机会。
- ❌ 错的：对"行业外部趋势"的敏感度不够。建筑行业数字化转型的政策推动是后来才知道的。
- ❌ 部分错：对转型期间的团队士气管理预估不足。非建筑行业的团队成员因为"看不到自己在公司未来的位置"而离开了。

**如果重来一次（带着当时的信息）：**
- 还是会选转型。
- 但会在转型前花2周专门做"团队沟通计划"。
- 会在决策中增加一个"外部趋势验证"步骤。

**可迁移的判断原则：**
1. **当数据中有明显的"离群值"，优先假设它是信号而非噪声——然后去验证，而不是用"平均"掩盖它。**
2. **小团队（<20人）不要做双轨制。选择聚焦可能错，但选择分散几乎一定平庸。**
3. **在还有12个月以上跑道时，更倾向选择"可逆的冒险"。跑道越短，越保守。**

## 五、常见错误与纠正

### 错误1：后见之明——用结果反推"我早就想到了"

这是决策卡最常见的毒药。你的大脑会自动改写记忆，让过去的决策看起来比实际更高明。

**典型症状：**
- "我选择A是因为我知道A会成功"（你当时不知道）
- 决策卡里没有任何犹豫、纠结和不确定

**校正方法：**
- 写决策卡时，先写下"当时我不知道的事"
- 在"可选方案"中，认真写出你拒绝的选项的**吸引力**

### 错误2：理性化——把情绪决策包装成理性分析

很多决策在发生的那一刻是直觉和情绪的产物，但写决策卡时我们倾向于给它补上一套完美的理性分析框架。

**校正方法：**
- 如果当时做决定的主要驱动力是直觉，就写"直觉"——然后补上"我的直觉可能基于什么？"
- 诚实地写"情绪状态"——焦虑时的决策和冷静时的决策质量不同
- **不完全理性的决策也可以是好决策**——关键是诚实地记录它

### 错误3：缺少"拒绝理由"

只说"我选了A"只完成了决策卡的一半。另一半是"我为什么拒绝了B和C"。

| ❌ 不完整的拒绝 | ✅ 完整的拒绝 |
|:---|:---|
| "B不好" | "B在短期内更安全，但我判断B的'安全'是假象——如果我们14个人的团队精力分散，两边的交付质量都会下降，6个月后可能两边客户的留存都会下降。B的风险不是立即的，是延迟的，但延迟的风险往往更大。" |

### 错误4：失败决策只归因于外部，成功决策只归因于自己

| | 失败决策 | 成功决策 |
|:---|:---|:---|
| ❌ 人的天性 | "外部环境不好" | "我判断准确" |
| ✅ 决策卡要求 | 承认自己可能判断错了什么 | 识别运气和外部因素的作用 |

### 错误5：只写"大决策"，不写"小决策"

不是只有"是否转型"这种级别的决策才值得写。很多最有价值的决策卡是关于**日常战术选择**的：

- "为什么我给这个客户的报价比正常低了20%？"
- "为什么我选择先和这个部门谈而不是那个部门？"
- "为什么这个项目我拒绝了？"

日常决策更容易被遗忘，但也更频繁地发生——写好它们，积累的是**判断力的肌肉记忆**。

## 六、决策卡的"完成标准"

一张合格的决策卡，应该能通过以下测试：

- [ ] **"事后验证"测试**：一个知道结果的人读这张卡，能不能理解"为什么在当时的信息下，这个选择是合理的"？
- [ ] **"情绪诚实"测试**：卡片中有没有写到当时的犹豫、焦虑或不确定？
- [ ] **"拒绝理由"测试**：被拒绝的选项有没有被认真对待？
- [ ] **"过程vs结果"测试**：有没有分别评估"决策逻辑对不对"和"结果好不好"？

## 七、决策卡的维护：决策链的编织

单张决策卡有价值，但**决策链**的价值是10倍以上。

当你积累了多张决策卡后，把它们串起来：

```
决策A（2024年3月：选择进入建筑行业）
    │
    └─→ 决策B（2024年9月：全面转型）
            │
            └─→ 决策C（2025年3月：是否融资）
                    │
                    └─→ 决策D（2025年9月：是否扩充到制造业）
```

决策链让你看到：
- 你的决策模式——你是越来越保守还是越来越激进？
- 上次决策的结果如何影响下次决策——有没有"矫枉过正"？
- 哪些决策是可逆的、哪些是不可逆的——你的"不可逆决策"命中率如何？

欢迎提交你用五卡体系记录的真实案例！

---

## 提交指南

### 案例要求

一个合格的社区案例应满足：

- ✅ 基于**真实经历**（可脱敏处理，但请注明）
- ✅ 包含**至少一张案例卡**（推荐包含决策卡和方法论卡）
- ✅ 包含**具体数字和时间**
- ✅ 标注**是否已脱敏处理**
- ✅ 在案例开头注明**行业/领域**

### 提交步骤

1. Fork 本仓库
2. 在 `examples/` 目录下创建新文件夹，命名格式：`行业-案例主题/`（如 `SaaS-客户续约谈判/`）
3. 将你的卡片放在该文件夹中（Markdown 格式）
4. 在下方"案例列表"中添加你的案例条目
5. 提交 Pull Request

## 案例模板

以下是一个完整的社区案例提交模板。你可以直接复制使用。

### 文件结构

```
examples/你的行业-案例主题/
├── README.md          # 案例总览
├── 01-案例卡.md        # 发生了什么
├── 02-决策卡.md        # 关键决策
└── 03-方法论卡.md       # 可复用方法（可选）
```

### 案例卡模板

```markdown
# 案例卡：[案例名称]

> **脱敏说明**：本案例基于真实事件，[客户名称/人名/金额]已脱敏处理。

## 基础信息
- **项目/事件名称**: 
- **时间**: 202X年X月 — 202X年X月
- **角色**: 
- **行业/领域**: 
- **状态**: ✅已完成 / 🔄进行中

## 背景与起点
[当时是什么情况，为什么会有这件事。写具体的，不要写空话。]

## 关键行动时间线
### 阶段一：[名称]（第X-Y天）
- 做了什么
- 发现了什么

### 阶段二：[名称]（第X-Y天）
- 做了什么选择
- 具体执行步骤
- 中间遇到的意外

### 阶段三：结果（第X-Y天）
- 最终发生了什么

## 数据总结
| 指标 | 变化 |
|:---|:---|
| [指标1] | [变化] |
| [指标2] | [变化] |

## 经验提炼
- **做对了什么**（3条）
- **踩过的坑**（至少1条）
- **下次会怎么做**
```
